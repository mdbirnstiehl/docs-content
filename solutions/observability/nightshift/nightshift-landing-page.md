# Nightshift landing page [nightshift-landing-page]

:::{note}
Nightshift is currently in private beta. Access is limited and by invitation only.
:::

The Nightshift landing page is your central hub for active Significant Events. It shows what Nightshift has detected across your streams, what [investigations](nightshift-investigations.md) are running, and what needs your attention — all in one place.

## Access the landing page [nightshift-landing-page-access]

Navigate to **Observability > Nightshift** in {{kib}}. If Nightshift is enabled on your cluster, the landing page opens and begins loading Significant Events.

If you don't see any events yet, Nightshift may still be running its initial KI extraction and rule generation. This can take a few minutes on first run. If your streams have sufficient data, events will surface automatically.

## Significant Events list [nightshift-events-list]

The landing page is divided into two panels:

- **Summary panel** — Shows the count of events that need action and resolved events, plus blast radius pills showing affected entities. Click a pill to filter the event list.
- **Event list panel** — Shows individual Significant Events sorted by most recently generated.

Each event in the list shows:

- **Name** — A concise title describing what happened, generated by Nightshift.
- **Time** — How long ago the event was generated.
- **Investigation status** — **Investigating** (animated, investigation in progress) or **Investigated** (complete).
- **Blast radius** — The entities affected by this event.

### Filter events [nightshift-events-filter]

Use the blast radius pills in the summary panel to filter the event list to a specific entity. The event list also separates events by status:

- **Needs action** — Events that are active and require attention.
- **Resolved** — Events that have cleared.

### View all events [nightshift-events-view-all]

Click **View all Significant Events** to open the full Significant Events management interface in the Streams app, where you can see the complete history, manage rules, bulk dismiss or promote events, and adjust settings.

## Significant Event flyout [nightshift-event-detail]

Click any event in the list to open the event flyout. The URL updates with an `eventId` query parameter, so you can share or bookmark the link to reopen the same flyout.

### Overview [nightshift-event-detail-overview]

The top of the flyout shows:

- **Title** — The name of the event.
- **Status** — **Needs action** (for open events) or no triage badge (for resolved events).
- **Investigation status** — **Investigating** or **Investigated**.
- **Timestamp** — When Nightshift created this event.
- **Summary** — A plain-language description of what Nightshift detected and why it's significant. Long summaries collapse with a **Show more** / **Show less** toggle.

### Detections [nightshift-event-detail-detections]

The **Detections** section lists the individual signals that contributed to this Significant Event. Each detection card shows:

- **Rule name** — The detection rule that fired.
- **Timestamp** — When the detection occurred.
- **Change point type** — The kind of change detected (for example, spike or dip).
- **Stream** — The data stream the detection came from.
- **Sparkline** — A mini chart shaped by the change point type, showing the detection pattern.

Click any detection card to open the detection flyout.

#### Detection flyout [nightshift-detection-flyout]

The detection flyout shows:

- **Rule name, detection badge, change point badge, and timestamp** — A summary of the detection.
- **Summary** — A description of the signal, with **Show more** / **Show less** for long text.
- **Associated entities** — Clickable entity chips. Click any entity to open the entity flyout.
- **Data trend** — A chart shaped by the change point type showing the detection pattern.
- **ES|QL query** — The query behind the detection. Click **Open in Discover** to explore the raw data, or copy it directly.

### Entities [nightshift-event-detail-entities]

Click any entity chip in the detection flyout to open the entity flyout. This shows what Nightshift knows about that entity based on extracted Knowledge Indicators:

- **Summary** — What the entity is and its role in your system.
- **Evidence** — The log data that supports the entity's identification.
- **Raw document** — The source document from which the entity was extracted.
- **Service badge** — Shown when the entity is identified as a service.

Click **Open in chat** from the entity flyout to start an AI conversation with the entity attached as context.

### Investigations [nightshift-event-detail-investigations]

The **Investigations** section shows the autonomous [investigation](nightshift-investigations.md) Nightshift ran for this event. Each investigation run appears as an accordion entry showing:

- **Status** — **Investigating** (live, streaming progress) or **Investigated** (complete).
- **Full investigation output** — For completed investigations: summary, hypotheses with confidence levels, conclusion, next steps, and any gaps found.

For more on what investigations produce and how they work, see [Investigations](nightshift-investigations.md).

## Chat about a Significant Event [nightshift-chat]

Every Significant Event has an **Open in chat** button, available from the event list, the event flyout, the detection flyout, and the entity flyout. Clicking it opens an AI Assistant conversation with the relevant context attached. You can:

- Ask for more detail about what happened and why.
- Request remediation options.
- Ask about affected services or dependencies.
- Explore hypotheses the investigation didn't fully cover.

The AI Assistant has access to the event's detections, associated KIs, [memory](nightshift-memory.md) pages about your system, and — if [Code Intelligence](nightshift-code-intelligence.md) is configured — your repository history.
