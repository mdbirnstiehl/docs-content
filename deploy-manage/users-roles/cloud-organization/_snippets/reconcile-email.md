Before July 2026, each organization required its own {{ecloud}} account and email address. You can now use one {{ecloud}} account across multiple organizations, and you might want to remove redundant accounts.

For each organization where a user currently signs in with a different email address, invite them again using the email address that you want them to use everywhere. Assign the same [roles and privileges](/deploy-manage/users-roles/cloud-organization/user-roles.md) they had on the alternate account.

After the user accepts the invitation, you can [delete](/deploy-manage/users-roles/cloud-organization/manage-users.md#manage-existing-users) their alternate account from that organization.
