You can control access to AutoOps in your connected cluster. During setup, you assign [user roles](/deploy-manage/users-roles/cloud-organization/user-roles.md#ec_connected_cluster_access_roles) so only users in your organization with the appropriate permissions can view cluster metrics, health events, and performance visualizations in AutoOps. To learn more, refer to the [privacy datasheet for AutoOps in Cloud Connect](https://www.elastic.co/pdf/elastic-privacy-datasheet-autoops-cloud-connect.pdf).
